Recursão

É um método de resolução de problemas que envolve quebrar um problema em subproblemas menores e menores até chegar a um problema pequeno o suficiente para que ele possa ser resolvido trivialmente. Em termos gerais, a recursão pode ser considerada como um processo de repetição de uma rotina. Logo, pode ser definida como uma rotina (função) que chama a si mesma.

Bem, se a rotina chama a si mesma inúmeras vezes, é preciso então tomar muito cuidado com o LOOP. Para que um LOOP não ocorra é necessário definir uma condição de terminação de forma correta. A função recursiva é dividida em duas partes: caso trivial e caso geral.
♦ Caso trivial: geralmente é o caso mais básico, fácil de resolver e retorna um resultado.
♦ Caso geral: esse novo problema é parecido com o original, a rotina chama uma nova cópia de si mesma para trabalhar no problema menor.
Exemplo:n!
CT(problema simples): 0! = 1 e 1! = 1
CG(problema complexo): n!=n*(n-1)

Quando é utilizada uma sequência recursiva, não é possível saber o valor do n-ésimo termo sem ter o valor do termo a(n-1), por isso é necessário que a função chame ela mesma.
Abaixo temos um exemplo prático em linguagem C de uma função fatorial recursiva:

int fat(int n){
if(n == 0){
return 1;
} else {
return n*fat(n-1);
}
}

Como podemos ver, a função fat(n) descrita vai receber um valor indicado por n e vai verificar se este valor é 0 para assim, retornar o valor 1. Caso o valor não seja 0, a função vai retornar n multiplicado pela própria função fat, porém com n-1 como argumento. Desta foma, a função será executada novamente para n-1 até chegar em n = 0, que vai finalizar a função multiplicando todos os resultados das chamadas dela, assim calculando o fatorial do número.

Abaixo temos outro exemplo de recursão em pseudo-código para divisão:

Função divisaoRec(inteiro num, inteiro den) retorna Inteiro
Inicio
Se (num < den)
Então
Função_Retorna(0)
Senão
Função_Retorna(divisaoRec(num-den, den) + 1)
Fim_Se
Fim

Tutorial

Neste software, é possível criar a sua própria sequência recursiva para se ter um auxílio para a resolução de exercícios de encontrar o n-ésimo termo da sequência. Também temos exercícios pré-definidos para serem resolvidos passo a passo.

Para acessar esses recursos, basta abrir o menu no canto superior esquerdo da página e selecionar a opção desejada.

Criar Função

Para a opção de Criar função, teremos inicialmente, um input para a quantidade de elementos. Em seguida, o valor de cada um desses elementos deve ser digitado nas caixas de input seguintes e por fim, escrevemos a função em si para a expressão indicada para todo a(n) maior que o valor do primeiro input.
Segue exemplo para fibonacci:
- Quantidade de elementos iniciais = 2
- a(1) = 1
- a(2) = 1
- Expressão recursiva = a(n-1)+a(n-2)

Regras Criar Função

- Você só tem 3 dicas, portanto use com sabedoria.
- Basta colocar a resposta para o termo atual para prosseguir.
- Em caso de acerto, sua nota aumenta.
- Em caso de erro, obviamente, sua nota diminui.
- Ao utilizar ajudas, sua nota será afetada negativamente.
- A nota é calculada da seguinte forma: acertos/(acertos+erros)-(dicas/3).

Exercícios

Para a opção dos exercícios predefinidos, basta escolher o exercício desejado na aba de seleção. Nesta opção, temos 3 caixas de input, em que se deve colocar os valores indicados pelos placeholders.

Regras Exercícios

- Você possui infinitas dicas, porém elas afetarão sua nota negativamente.
- Para avançar, basta responder o que for pedido nas 3 caixas.
- A nota aumenta caso esteja certa e diminui caso contrário.
- A nota é calculada da seguinte forma: acertos/(acertos+erros) - dicas/(acertos/(acertos+erros))
